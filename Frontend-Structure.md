
## DriftShield Frontend Structure

  

>  **Note:** The  `reports/` directory is mounted from an AWS EBS (Elastic Block Store) volume. This folder is not version-controlled and is excluded from GitHub.
>
>  **EBS (Elastic Block Store)** is used as persistent storage for drift reports.
---

```

driftshield-frontend/
│
├── .github/   # CI/CD via Github Actions
│ 	└── workflows/
│ 	     └── ci_cd.yml
│
├── reports/                      # Mounted with AWS EBS Volume
│    └── drift_report.html        # Drift reports generated by the backend
│
├── config/
│ 	 └── config.py
│
├── pages/
│    └── 1_Prediction.py       # Prediction & AI Insights Page
│    └── 2_Drift_Detection.py  # Drift Monitoring & AI Insights Page
│    └── 3_Model_Registry.py   # Model Registry Management Page
│
├── utils/
│    └── api_client.py     	   # Handles all backend API requests
│    └── helpers.py			   # UI helper functions
│    └── sample_data.py        # Generates sample prediction inputs
│
├── Overview.py    # Landing page and project overview
│
├── Dockerfile
│
├── README.md 	   # Project documentation
│
├── Frontend-Structure.md   # Frontend directory structure
│
├── requirements.txt  # All necessary packages for this application
│
├── .dockerignore
│
├── .gitignore
│
├── .env # (excluded from GitHub)

```

| Directory | Purpose |
